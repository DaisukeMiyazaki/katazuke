# Katazuke

過去の自分が残した、リンクが肥大したノートと向き合うための Obsidian プラグイン。

Obsidian 1.8.7 以上が必要。[コミュニティプラグイン一覧](https://community.obsidian.md/plugins/katazuke)から、設定 → コミュニティプラグイン → 閲覧で「Katazuke」を検索して導入できる。

## 考え方

- 結合しすぎたノートは不健全な兆候になりうる。ただしグラフビューでは見つけにくい。
- 直接測るべきは次数（リンク数）。放置された古いノートほど整理の必要が高いとみなし、次数 × 鮮度で採点する。
- 意図的なハブ（目次ノート）は tag で除外し、無自覚に膨らんだノートだけを残す。

採点式:

```
score = degree * (1 + ageDays / freshnessHalfLifeDays)
```

`degree` は被リンク数 + 発リンク数。半減期だけ放置されると採点が 2 倍になる。

## 2つの作法（モード）

- 一件と向き合う: 首位の 1 件だけを提示する軽い整理。日課や隙間時間向け。
- 数件と向き合う: 上位 N 件を提示する本格的なセッション。腰を据えて整理する用途。

## たまりの山

リンク過多とは逆の失敗、どこにも繋がらず放置されたノート（たまり）と向き合うためのレンズ。「たまりの山を開く」コマンドかリボンの山アイコンで、右サイドパネルに開く。

- 孤立していて一定日数（既定 90 日）触っていないノートが粒になり、砂山として積み上がる。古い粒ほど濃く、山の芯に沈む。
- 一山は「一山の粒数」（既定 10）＝一度に崩し切れる量。積むのは最も放置の古い一角から。
- 粒をクリックすると判定。ノートは山の横に（フォーカスを奪わずに）開き、粒は砂けむりになって消え、上の粒が落ち、急な斜面は雪崩れる。色つきの粒は「ここから」の道しるべ。
- 崩し切ると残り数が示され、クリックで次の山が立つ。
- 判定のたびに保存庫内の記録ファイルへ 1 行追記され、設定画面に合計と直近 30 日の棒グラフで「片付けの軌跡」が描かれる。記録は保存庫のファイルが原本なので、同期すれば端末をまたいで軌跡がつながる。

## 設定

- ハブ除外タグ（既定 `hub`）
- 鮮度の半減期（日、既定 90）
- 数件モードの件数（既定 7）
- 最小次数（既定 5）
- 一山の粒数（既定 10）
- たまりとみなす放置日数（既定 90）
- 山の配色（既定 Obsidian に合わせる。ライト / ダーク固定も可）
- アクセントカラーを使う（既定 オフ。判定候補の粒を Obsidian のアクセントカラーで描く）
- 対象外フォルダー（配下のノートを山にも候補にも含めない。ドロップダウンで追加）
- 記録ファイル（既定 `katazuke-log.md`）

## アクセスするもの

完全にオフラインで動作する。

- vault 内の全ノートのパス、解決済みリンクグラフ、各ノートの最終更新時刻、タグを読む（採点と山の抽出に必要）。
- 通信は一切しない。vault の外へ何も送らない。
- クリップボードは読み書きせず、既存のノートは改変しない。
- 書き込むのは 2 つだけ。自身の設定（Obsidian のプラグインデータ API 経由）と、たまりの山で判定したときに記録ファイル（既定 `katazuke-log.md`、変更可）へ追記する 1 行。

## 開発

```
npm install
npm test        # vitest（純ロジックのテスト）
npm run typecheck
npm run lint    # eslint-plugin-obsidianmd（公式レビューを再現）
npm run build   # main.js を生成
```

Obsidian で試すには、`manifest.json` / `main.js` / `styles.css` を vault の
`.obsidian/plugins/katazuke/` に置いて有効化する。
